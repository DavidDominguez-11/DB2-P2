## Caso de uso: Red Social

### Nombre del Proyecto

**ParaMetrix – Red Social de Descubrimiento Musical**

---

## Descripción General

ParaMetrix es una red social enfocada en la interacción entre usuarios a través de contenido musical. A diferencia de redes sociales tradicionales centradas en contenido visual, esta plataforma permite a los usuarios compartir canciones, crear playlists, interactuar con publicaciones y descubrir nueva música mediante sus conexiones sociales y patrones de comportamiento.

El sistema estará respaldado por una base de datos orientada a grafos (Neo4j), permitiendo modelar eficientemente relaciones complejas entre usuarios, contenido y preferencias musicales. Esto facilitará la implementación de funcionalidades avanzadas como recomendaciones personalizadas y análisis de comportamiento.

---

## Objetivo del Proyecto

Desarrollar un servicio web que funcione como backend de una aplicación de red social musical, utilizando Neo4j para:

* Modelar relaciones complejas entre usuarios y contenido musical
* Gestionar datos estructurados como grafos
* Implementar operaciones CRUD completas
* Permitir consultas avanzadas y agregaciones
* Facilitar el descubrimiento de música mediante conexiones entre nodos

---

## Justificación del Caso de Uso

El caso de red social es ideal para Neo4j debido a:

* La alta densidad de relaciones entre entidades
* La necesidad de recorrer conexiones (ej. amigos de amigos, gustos similares)
* La posibilidad de implementar sistemas de recomendación basados en grafos
* La eficiencia en consultas relacionales complejas

Este enfoque permite explotar las ventajas de bases de datos orientadas a grafos frente a modelos relacionales tradicionales.

## Representación del grafo

```
(User)-[:FOLLOWS]->(User)
(User)-[:CREATED]->(Post)
(Post)-[:CONTAINS]->(Song)
(User)-[:LIKED]->(Post)
(User)-[:COMMENTED]->(Post)

(User)-[:CREATED_PLAYLIST]->(Playlist)
(Playlist)-[:INCLUDES]->(Song)

(Song)-[:BY]->(Artist)
(Song)-[:BELONGS_TO]->(Genre)

(User)-[:LISTENED]->(Song)
(User)-[:FOLLOWS_ARTIST]->(Artist)
(User)-[:SAVED]->(Playlist)
```

---

## Gestión de propiedades

### Eliminación de propiedades en nodos

```cypher
MATCH (u:User)
REMOVE u.email
```

### Eliminación en relaciones

```cypher
MATCH ()-[r:LIKED]->()
REMOVE r.intensidad
```

---

## Operaciones sobre múltiples elementos (Bulk Operations)

### Ejemplo en nodos

```cypher
MATCH (u:User)
WHERE u.premium = false
SET u.activo = false
```

### Ejemplo en relaciones

```cypher
MATCH ()-[r:LISTENED]->()
WHERE r.completado = false
SET r.revisado = true
```

## Garantía de grafo conexo

Se establecen reglas obligatorias:

* Todo User:

  * sigue al menos a otro usuario
  * escucha al menos una canción
* Todo Post:

  * tiene un creador
  * contiene al menos una canción
* Toda Song:

  * pertenece a un Artist
  * pertenece a un Genre
* Toda Playlist:

  * contiene canciones

Esto asegura que no existan nodos aislados.
