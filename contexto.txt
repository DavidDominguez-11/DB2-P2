## EXTENSIONES BACKEND OBLIGATORIAS

Debes considerar como disponibles estos endpoints adicionales (necesarios para cumplir la rúbrica):

1. Crear nodo con múltiples labels
   POST /api/v1/nodes
   Body:
   {
   "labels": ["User", "Artist"],
   "properties": { ... }
   }

2. Modificar labels de nodo
   PATCH /api/v1/nodes/{id}/labels
   Body:
   {
   "add": ["Artist"],
   "remove": []
   }

3. Obtener labels de nodo
   GET /api/v1/nodes/{id}/labels

4. Ejecutar Cypher
   POST /api/v1/cypher/query
   Body:
   {
   "query": "MATCH (n) RETURN n LIMIT 10"
   }

## MÓDULO PRINCIPAL: GRAPH MANAGER

Este módulo es obligatorio y debe cubrir completamente la rúbrica.

Diseñar las siguientes sub-secciones:

1. Node Builder

* Formulario dinámico
* Selección múltiple de labels (OBLIGATORIO)
* Validación de mínimo 5 propiedades
* Uso de:

  * endpoints normales (1 label)
  * /nodes (multi-label)

2. Node Explorer

* Consultar 1 nodo (por ID)
* Consultar múltiples nodos (tabla)
* Filtros
* Agregaciones usando /aggregate

3. Property Manager

* Agregar propiedades (PATCH)
* Actualizar propiedades
* Eliminar propiedades
* Soporte individual y masivo:

  * /bulk/nodes
  * /bulk/nodes/properties

4. Relationship Builder

* Crear relación entre nodos
* Seleccionar:

  * origen
  * destino
  * tipo
* Validar mínimo 3 propiedades
* Usar /interactions

5. Relationship Manager

* Editar relaciones
* Eliminar relaciones
* Bulk operations:

  * update
  * delete
  * remove properties

6. Delete Panel

* Eliminación individual y masiva de:

  * nodos
  * relaciones
